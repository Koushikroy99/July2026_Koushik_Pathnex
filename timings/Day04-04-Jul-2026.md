# 🔐 Linux File Permissions

Linux file permissions control who can read, write, or execute a file or directory.

## Permission Format

```text
drwxr-xr--
```

| Symbol | Meaning |
|---------|---------|
| d | Directory |
| r | Read Permission |
| w | Write Permission |
| x | Execute Permission |
| - | Permission Not Granted |

## Permission Categories

- User (Owner)
- Group
- Others

## Numeric Permission Values

| Permission | Value |
|------------|------:|
| rwx | 7 |
| rw- | 6 |
| r-x | 5 |
| r-- | 4 |
| --- | 0 |

## Common Permission Examples

| Permission | Meaning |
|------------|---------|
| 755 | rwxr-xr-x |
| 644 | rw-r--r-- |
| 700 | rwx------ |
| 777 | rwxrwxrwx |

## chmod Command

Used to change file or directory permissions.

```bash
chmod 755 file.txt
chmod 644 file.txt
chmod +x script.sh
```

# 👥 User & Group Management
'
Linux supports multiple users and groups for secure access control.

## Common Commands

| Command | Purpose |
|---------|---------|
| `useradd` | Create a new user |
| `groupadd` | Create a new group |
| `chown` | Change file owner |
| `chgrp` | Change group ownership |
| `usermod -aG` | Add a user to a group |
| `umask` | Set default file permissions |

## Examples

```bash
sudo useradd devuser
sudo groupadd devops
sudo usermod -aG devops devuser

sudo chown devuser file.txt
sudo chgrp devops file.txt

umask
```

# 📦 File Compression & Archiving

## Common Commands

| Command | Purpose |
|---------|---------|
| `tar` | Create or extract archive files |
| `gzip` | Compress files |
| `zip` | Create ZIP archives |
| `ln` | Create symbolic links |
| `find` | Search files and directories |

## Examples

### Create a tar archive

```bash
tar -cvf backup.tar project/
```

### Extract a tar archive

```bash
tar -xvf backup.tar
```

### Compress a file

```bash
gzip backup.tar
```

### Create a ZIP archive

```bash
zip backup.zip project/
```

### Create a symbolic link

```bash
ln -s /home/user/file.txt shortcut
```

### Find a file

```bash
find /home -name "*.txt"
```

# ⏰ Cron Jobs (Task Automation)

Cron is used to automate repetitive tasks such as backups, log cleanup, monitoring, and scheduled scripts.

## Edit Crontab

```bash
crontab -e
```

## Crontab Syntax

```text
* * * * * command_to_execute
│ │ │ │ │
│ │ │ │ └── Day of Week (0–7) (0 or 7 = Sunday)
│ │ │ └──── Month (1–12)
│ │ └────── Day of Month (1–31)
│ └──────── Hour (0–23)
└────────── Minute (0–59)
```

## Meaning of Each Field

| Field | Description | Example |
|-------|-------------|---------|
| Minute | Minute when the command runs | 30 |
| Hour | Hour when the command runs | 9 |
| Day of Month | Date of the month | 15 |
| Month | Month | 7 (July) |
| Day of Week | Day | 1 (Monday) |

- Every month

---

## Easy Trick to Remember

Read from left to right.

```text
Minute → Hour → Day of Month → Month → Day of Week → Command
```

Shortcut:

```text
M H D M W
│ │ │ │ │
│ │ │ │ └── Week
│ │ │ └──── Month
│ │ └────── Date
│ └──────── Hour
└────────── Minute
```

Remember:

- `*` = Every
- Number = Specific value

Examples:

```text
* * * * *     → Every minute
0 2 * * *     → Every day at 2:00 AM
30 8 * * 1    → Every Monday at 8:30 AM
30 9 15 7 1   → 9:30 AM on 15 July if it is Monday
```
